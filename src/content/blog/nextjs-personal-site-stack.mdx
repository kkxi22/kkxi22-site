<h2 id="why-now">为什么现在重构</h2>

旧站点只是两个静态占位页，能用，但没有内容模型，也没有一个适合长期维护的结构。个人介绍、博客和项目展示分别散落时，后续每次改版都会重复处理路径、样式和 SEO。

<Callout tone="success">
这次重构的目标不是做一个“看起来更新”的首页，而是把未来两三年的公开内容基础打稳。
</Callout>

<h2 id="stack-choice">技术栈选择</h2>

选择 `Next.js 16 + TypeScript + Tailwind CSS 4 + MDX` 的原因很直接：

- App Router 足够稳定，静态导出能力成熟。
- TypeScript 让内容数据、导航结构和动态路由都能保持约束。
- Tailwind 4 让样式系统能快速贴近目标站点的细节密度。
- MDX 适合写博客，也适合写项目说明页。
- `next/font`、`next/image`、Metadata 和 OG 都是内建能力，不需要再额外拼装。

```tsx
export const metadata = {
  title: "为个人站点选定 Next.js 16 技术栈",
};
```

<h2 id="delivery">交付方式</h2>

这个站点最终以静态导出方式发布到 `kkxi22.cn`，原因是路径稳定、部署简单，也方便和现有的 `nginx` 配置衔接。公开页面全部走静态文件，私有服务仍通过 `/openclaw/` 反向代理处理。

这样做的结果是：

1. 公开层不依赖常驻 Node 进程。
2. 内容更新只需要重新构建并同步导出目录。
3. 域名表面保持稳定，私有入口继续保留边界。
