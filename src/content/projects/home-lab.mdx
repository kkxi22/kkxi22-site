<h2 id="lab-goal">目标</h2>

家庭服务器最常见的问题不是服务不够多，而是入口太碎。主域名、博客、面板、实验页各自分散时，维护成本会快速变高。

<h2 id="surface">站点界面</h2>

Home Lab Surface 的做法是：

- 主域名只保留长期公开内容。
- 私有服务继续通过受保护路径暴露。
- 所有公开页面采用同一套文档式设计语言。

这样一来，站点本身就变成了整个 Home Lab 的索引层。

<h2 id="deployment">部署</h2>

部署链路刻意保持简单：

```bash
npm run build
rsync -a out/ /opt/homebrew/var/www/kkxi22/site/
```

`nginx` 只负责三件事：服务静态导出目录、保留 `/openclaw/` 代理、维持主域名稳定。只要这三件事不被破坏，后续换内容、换样式甚至换项目结构都不会太痛苦。
